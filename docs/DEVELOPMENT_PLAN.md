# UMax Admin 当前开发计划（基于代码现状）

> 本文档聚焦三件事：当前问题与隐患、未完成/开发中能力、按优先级的实施路线。  
> 架构与实现细节见 `docs/PROJECT_ARCHITECTURE.md`。

**文档版本**: 2.0.0  
**最后更新**: 2026-04-30

---

## 1. 现状速览

### 1.1 已形成的基础能力

- 已具备 Vue 3 + TypeScript + Vite+ + Naive UI 的中后台基础骨架。
- 登录鉴权主链路可用：账号密码登录、双 token 存储、401 自动续期、路由守卫与权限拦截。
- 布局主题系统可用：`vertical` / `sidebar` / `horizontal` 三种布局、亮暗主题、主色和圆角、顶栏/侧栏区域主题。
- 路由/菜单/面包屑/标签页联动可运行，偏好设置可动态驱动布局与主题配置。
- Mock 与真实 API 共存的联调能力可用，但生产策略仍需收口。

### 1.2 当前总体判断

- 项目已达到“模板级可运行”，但距离“业务级可交付”还有明显差距。
- 当前最关键不是继续堆功能，而是先收敛生产风险、补齐核心闭环、建立最小质量闸门。

## 2. 主要问题与隐患清单（与第 4 节一一对应）

### P0（高风险，需优先处理）

- **R1 / P0-1 生产误用 Mock 风险**
  - 现状：生产构建可能注入 Mock。
  - 风险：线上接口行为不可控。
  - 对应执行项：`4.1 -> P0-1`

- **R2 / P0-2 文档口径与实现不一致**
  - 现状：部分文档把未闭环功能标记为已完成。
  - 风险：误导验收范围、错误评估交付状态。
  - 对应执行项：`4.1 -> P0-2`

- **R3 / P0-3 安全与可观测性细节不规范**
  - 现状：登录链路存在调试日志、默认演示凭据。

  - 对应执行项：`4.1 -> P0-3`

**完成标准**

- 用户相关路由可演示真实交互闭环。
